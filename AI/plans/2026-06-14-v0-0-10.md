# v0.0.10 Release — Lossless polish: ByParts drift + disk fast-path
**Status: DRAFT — awaiting approval**
**Date: 2026-06-14**

## Goal

Resolve the two deferred issues from the v0.0.7 release cycle: implement a stream-aligned split snap algorithm (correcting the ByParts drift issue) and provide a storage-efficient direct merge fast-path (bypassing caching phases when physical paths resolve and the reliability flag is disabled).

## Scope (in)

1. **Audio-Aligned ByParts Snap Fix**:
   - Query audio packet PTS lists on split starts using a new engine function: `ffprobe -v error -hide_banner -select_streams a:0 -show_packets -show_entries packet=pts_time -of csv=p=0 <uri>`.
   - Update [CutPlanner](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/domain/splitter/CutPlanner.kt) to substitute `nearestKeyframeAtOrBefore(t)` logic with `nearestSafeBoundary(t)` taking both video keyframes and audio packet PTS lists.
   - Set cut bounds using `C = min(lastVideoKeyframe, lastAudioPacketPTS)` ensuring both streams have packet boundaries at the cut.
   - Adjust [Splitter](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/domain/splitter/Splitter.kt) to request and pass this stream information.

2. **Disk Optimization Fast-Path**:
   - Check if physical file paths can resolve for all parts using [MergePathResolver](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/domain/merger/MergePathResolver.kt).
   - If *every* input file and the target output directory can be mapped to physical file paths (direct storage paths), and the settings preference `improveReliability` is OFF, then execute a direct merge.
   - Skip the copy-to-cache (Staging) and copy-from-cache (Copying-to-output) phases entirely. Execute the FFmpeg concat command mapping directly from original inputs to the final output file destination path.
   - Keep the staging fallback if *any* file path fails to resolve to a direct physical path, or if `improveReliability` is ON.
   - Adjust `getOverallPct` calculations inside [Merger](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/domain/merger/Merger.kt) to maps progress values based on active stages (e.g. Concat spanning 0-100% when staging/copying are bypassed).

## Out of scope

- UI/Settings screen design modifications.
- Room database scheme updates.
- Changes to manifest configurations.
- Non-media folder validations.

## Acceptance criteria (measurable)

1. **Drift Reduction**: Running a ByParts 3 split on the 3:44:07 reference file produces three parts whose combined durations sum to <= 3:44:11 (compared to 3:44:12+ previously).
2. **Direct Execution**: Merging with `improveReliability = OFF` skips cache operations (verify `cacheDir` size remains under 100MB throughout the run).
3. **Correct Progress Ranges**: Direct merger runs show smooth progress bar transitions spanning the full 0% to 100% range.
4. **Fallback Safety**: Forcing a non-resolvable URI content schema reverts the merger to safe staged operations automatically.
5. **No Regressions**: All standard tests continue to pass.

## Rollback plan

We can easily revert to v0.0.9 behaviors by toggling the `improveReliability` flag to ON via the Settings screen. The cut planning edits can be reverted to use `nearestKeyframeAtOrBefore` if any audio desynchronization issues are reported.
