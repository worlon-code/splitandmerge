# v0.0.8 Release — Settings + Update + Folder validation + Diagnostics log
**Status: DRAFT — awaiting approval**
**Date: 2026-06-14**

## Goal

Make Settings real (migrate from SharedPreferences to Jetpack Preferences DataStore), ship a real working update check mechanism with SHA-256 validation of the downloaded APK, stop silently accepting invalid output storage folders in the UI via a dedicated validator class, and implement an in-app diagnostic log viewer with a share action.

## Scope (in)

1. **SettingsRepository**:
   - Create a Hilt-singleton [SettingsRepository](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/data/settings/SettingsRepository.kt) wrapping `androidx.datastore.preferences.core.Preferences`.
   - Persist values for keys: `themeMode` (enum), `defaultCapGb` (float/double), `improveReliability` (boolean), `keepScreenOn` (boolean), and `defaultOutputFolderUri` (string).
   - Expose a `Flow<SettingsState>` containing all values, and suspend setters to update each preference.
   - Migrate [SettingsViewModel](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/ui/settings/SettingsViewModel.kt) off `SharedPreferences` to use this new repository.

2. **Battery Optimization Opt-in**:
   - Wire the row in [SettingsScreen](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/ui/settings/SettingsScreen.kt) to check current battery optimization whitelist state using `PowerManager.isIgnoringBatteryOptimizations()`.
   - If not ignoring, trigger the `ACTION_REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` system intent via `startActivity`. Reflect the updated state in the view model when returning to the app.

3. **Update Check Flow**:
   - Use Retrofit + OkHttp client (dependencies to be added to Gradle configuration).
   - Fetch a raw update manifest JSON matching `VersionInfo` data class (keys: `latest`, `versionCode`, `url`, `size`, `sha256`, `minSdk`, `publishedAt`, `changelog`).
   - Download the APK to `cacheDir/update/app-update.apk` on demand, and check the downloaded file's SHA-256 hash.
   - If the SHA-256 hash matches, invoke a `PackageInstaller` session to trigger system-level installation. Guard this step: if `BuildConfig.DEBUG`, show a toast warning `"Install disabled in debug build"` and skip package installation.
   - Fail-safe check: Mismatched SHA-256 must abort, log a warning, and delete the downloaded file immediately.

4. **Output Folder Validation**:
   - Create [OutputFolderValidator](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/domain/storage/OutputFolderValidator.kt) as a Hilt `@Singleton`.
   - Implement fail-fast validation checks in order:
     a) Tree URI reachable (check `DocumentFile.fromTreeUri(...) != null` and query content resolver).
     b) Permission persistent (ensure `contentResolver.persistedUriPermissions` has read and write permissions for the selected URI).
     c) Folder writable (attempt to write a temp probe file `"<uuid>.probe"` and delete it immediately).
     d) Space check: Query `StorageVolume.allocatableBytes` (or fall back to `usableSpace`) to verify that the remaining space is greater than or equal to `max(defaultCapGb, source-size-if-known)`.
   - Map validation states to sealed results: `Ok`, `NotReachable`, `PermissionRevoked`, `NotWritable(reason)`, `InsufficientSpace(needed, have)`.
   - Integrate with the folder pickers on [SettingsScreen](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/ui/settings/SettingsScreen.kt), [SplitConfigScreen](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/ui/splitconfig/SplitConfigScreen.kt), and [MergeConfigScreen](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/ui/mergeconfig/MergeConfigScreen.kt). Show a Material 3 `AlertDialog` on failure with options to retry or pick a different folder.

5. **Diagnostics Log Viewer**:
   - Implement [FileLoggingTree](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/domain/logging/FileLoggingTree.kt) extending `Timber.Tree` that writes daily logs to `context.cacheDir/logs/app-<yyyy-MM-dd>.log`.
   - Automatically purge files older than 7 days on application start.
   - Redact all occurrences of sensitive SAF URIs in log entries to `"tree://<host>/<authority>/<doc-id-hash>"` (hashing document IDs using the first 8 characters of SHA-256).
   - In Settings, expose a "View logs" option leading to a pageable list of logs newest-first ([LogViewerScreen](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/ui/settings/LogViewerScreen.kt)).
   - Provide a top-app-bar action to share the selected log file using `ACTION_SEND` and a `FileProvider` URI mapping to the daily log directory.

## Acceptance criteria (measurable)

1. **Persistent Preferences**: Changing theme modes or size defaults survives process destruction and restarts.
2. **Battery Opt-In**: Whitelist button requests permission, updating correctly on activity resume.
3. **Valid update detection**: Serving a mock update manifest updates settings card correctly between "Up to date" and "Available".
4. **Verification Failure Safe-Guard**: Forcing a checksum mismatch deletes downloaded update APKs immediately and throws error UI toast warnings.
5. **No Silent folder failure**: Pickers showing SD failures, missing permissions, or low space alert users explicitly, requiring retry actions.
6. **Redacted logs**: System log files must never print full content URI schemes.
7. **No Regression**: Engine splits and merge speeds matching v0.0.7 baseline ± 1s.

## Rollback plan

Preferences flow can fall back to SharedPreferences by restoring old bindings inside `AppModule` and changing ViewModel injection parameters. Added update modules and output validators are additive and can be stripped by removing their references in layout bindings.
