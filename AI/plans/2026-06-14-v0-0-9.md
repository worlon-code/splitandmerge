# v0.0.9 Release — Cleanup patterns + OSS notices + Branding & motion
**Status: DRAFT — awaiting approval**
**Date: 2026-06-14**

## Goal

Replace mock data in Cleanup Patterns and OSS Notices screens with real database persistence and packaged JSON assets respectively. Introduce unified app branding including vector-only adaptive launcher icons and an Androidx-native Splash Screen. Provide polished custom Compose-native loading animations for long-running operations.

## Scope (in)

1. **Room Persistence for Cleanup Patterns**:
   - Create a new Room entity [CleanupPatternEntity](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/data/db/entity/CleanupPatternEntity.kt) with attributes: `id` (String Primary Key), `regex` (String), `replacement` (String), `caseInsensitive` (Boolean), `enabled` (Boolean), `orderIdx` (Int), `label` (String), `isBuiltIn` (Boolean), and `createdAt` (Long).
   - Create [CleanupPatternDao](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/data/db/CleanupPatternDao.kt) supporting CRUD operations, status toggles, ordering indices, and returning observable `Flow<List<CleanupPatternEntity>>`.
   - Bump `AppDatabase` version `3 → 4`. Add [Migration_3_4](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/data/db/migrations/Migration_3_4.kt) to execute SQLite commands creating the table and pre-seeding the 12 default patterns listed in specs.
   - Refactor [CleanupPatternsViewModel](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/ui/cleanup/CleanupPatternsViewModel.kt) to observe `CleanupPatternDao` flows instead of mock data lists.
   - Update [TitleCleaner](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/domain/cleanup/TitleCleaner.kt) to read patterns dynamically from database flows, keeping a memory cache of the snapshot for synchronous cleaning runs. Keep a fallback hardcoded list if database flows are uninitialized.

2. **Asset-Backed OSS Notices**:
   - Package a JSON license file at `app/src/main/assets/oss-licenses.json` mapping dependencies to properties: `name`, `publisher`, `license`, `licenseText`, `url`.
   - Populated with at least 9 entries covering core framework libraries: FFmpegKit LGPL, Jetpack Compose, Hilt, Room, kotlinx.serialization, Retrofit, OkHttp, Timber, and AndroidX Core.
   - Implement [OssNoticesViewModel](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/ui/oss/OssNoticesViewModel.kt) to read, parse, and emit license configurations on `Dispatchers.IO` using `kotlinx.serialization`. Rebuild [OssNoticesScreen](file:///d:/Repos/splitandmerge/app/src/main/kotlin/com/splitandmerge/mkvslice/ui/oss/OssNoticesScreen.kt) to consume this flow.

3. **App Branding & Visual Polish**:
   - **Adaptive Launcher Icons (Vector Only)**:
     - Foreground mark ([ic_launcher_foreground.xml](file:///d:/Repos/splitandmerge/app/src/main/res/drawable/ic_launcher_foreground.xml)) — stylised film strip cut motif (~108dp outer, ~66dp safe zone).
     - Background layer ([ic_launcher_background.xml](file:///d:/Repos/splitandmerge/app/src/main/res/drawable/ic_launcher_background.xml)) — solid color matching primary container.
     - Monochrome variant ([ic_launcher_monochrome.xml](file:///d:/Repos/splitandmerge/app/src/main/res/drawable/ic_launcher_monochrome.xml)) for Android 13+ themed icons.
     - Remove legacy mipmap PNG raster files.
   - **androidx.core:core-splashscreen Integration**:
     - Define `Theme.App.Splash` in `themes.xml` wrapping the adaptive icon launcher drawable.
     - Launch `installSplashScreen()` in `MainActivity` before UI content composition. Defer showing content until initial folder checks and directory setup complete.
   - **Loading Motion**:
     - Create Compose-native `LoadingArc` (continuous scaling arc) inside `ui/components/` for probe and update checks.
     - Create `PulseDot` (continuous pulsing dots) for running state indicator icons in list cells.
     - Create shimmer loading placeholders (skeleton rows) for initial states on notice and log pages.
     - C.4: Wire LoadingArc to MergeOrderScreen during addParts() ffprobe pass (per K-017). Show indeterminate spinner with text "Verifying parts..." instead of letting the UI sit idle. Hide once addParts completes.

## Acceptance criteria (measurable)

1. **Room Initial State**: Fresh install contains exactly 12 built-in, pre-populated, enabled patterns.
2. **Persistent Pattern Updates**: Custom regex pattern additions and updates survive restarts.
3. **Cleaning Bypass**: Disabling "resolution patterns" filters skips them during title formatting passes.
4. **Protected Built-Ins**: Tapping delete on built-in rows displays error prompts explaining they cannot be removed.
5. **Database Preservation**: Upgrading databases from version 3 does not corrupt existing files.
6. **Licensing items**: License notices display at least 9 valid libraries.
7. **Vector scaling**: Launcher icons scale nicely across different notification/display contexts without pixel distortion.

## Rollback plan

Migrations can roll back by reverting database structures, dropping tables, and falling back to built-in clean regex arrays in memory. Asset notices can be removed easily by deleting the `oss-licenses.json` file.
