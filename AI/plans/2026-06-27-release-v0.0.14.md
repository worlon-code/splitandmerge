# Release v0.0.14 – Finalize Docs and Fix Update Check

## Problem
- The usage guide and README contain real movie titles and piracy site domains which must be replaced with neutral placeholders.
- `.gitignore` mistakenly excluded `docs/screenshots/`, breaking image rendering on GitHub.
- The `videosplitter-version.json` manifest generated by `deploy_release.ps1` includes a UTF‑8 BOM, causing a JSON parse error on the device (update check fails in v0.0.13 and v0.0.14).
- Release v0.0.14 is tagged but not verified after the manifest fix.

## Proposed change
1. **Docs cleanup**
   - Replace all real movie titles / site domains in `docs/USAGE.md` and `README.md` with generic placeholders while keeping a visible "before" example.
   - Ensure screenshots are committed and visible (already fixed `.gitignore`).
2. **Manifest fix**
   - Modify `deploy_release.ps1` to write `videosplitter-version.json` using UTF‑8 without BOM.
   - Regenerate the manifest with correct SHA‑256 and size for the existing `app-release.apk`.
3. **Verification**
   - Run `./gradlew lint` and `./gradlew test`.
   - Assemble the release APK (`./gradlew assembleRelease`).
   - Install the APK on the connected device, trigger "Check for updates", capture logcat, ensure no JSON parse error.
   - If the update check succeeds, no version bump is needed; otherwise, iterate.
4. **Release steps**
   - Ensure the existing Git tag `v0.0.14` points to the new commit (force‑push acceptable as no new version bump). 
   - Run `deploy_release.ps1` to upload the corrected manifest and APK to the same repository (no separate releases repo).
   - Update `AI/CHANGELOG.md` entry if any new fixes were added.
