# Fix ffprobe JSON crash and add unit tests

## Problem
- Large files (~59 GB) or files with E‑AC3 audio cause ffmpeg‑kit to interleave libav log lines into the ffprobe JSON output. This leads to `kotlinx.serialization` throwing "Unexpected JSON token ... got [".
- The existing implementation parses the raw session output directly, which is fragile.
- There may be an older Mockito‑based `ProcessFfprobeEngineTest` lingering in the test sources, which would cause duplicate class definitions.

## Proposed change
1. **Overwrite** `app/src/main/kotlin/com/splitandmerge/mkvslice/engine/impl/ProcessFfprobeEngine.kt` with the exact content supplied by the user. This implementation writes ffprobe JSON to a temporary file via `-o` and falls back to sanitising session output if needed.
2. **Overwrite** `app/src/test/kotlin/com/splitandmerge/mkvslice/engine/impl/ProcessFfprobeEngineTest.kt` with the exact test file supplied by the user.
3. **Delete any other** `ProcessFfprobeEngineTest` files under `app/src/test/` or `app/src/androidTest/` that may have been generated by previous runs (e.g., Mockito‑based versions).
4. Run **`./gradlew testDebugUnitTest --rerun-tasks`** and **`./gradlew lintDebug`** and capture the full (non‑cached) output. Verify that all 5 test cases execute and the build succeeds.
5. Build a **debug APK** (`./gradlew assembleDebug`).
6. Using the configured ADB path, install the debug build on the connected device, then manually open three file types:
   - a file with E‑AC3 audio,
   - the ~59 GB Matroska file,
   - a freshly merged E‑AC3 output.
   Verify via `adb logcat` that no "Failed to probe" or "Failed to load merge result" messages appear.
7. **Do not** bump version, commit, or modify any other files (including AI docs, `deploy_release.ps1`, `videosplitter-version.json`).
