# v0.0.8 Step 5 Implementation Plan — Folder Picker Validation Wiring

## Proposed Changes

### 1. New Composable Component: `FolderValidationDialog.kt`
- Location: `app/src/main/kotlin/com/splitandmerge/mkvslice/ui/components/FolderValidationDialog.kt`
- Stateless Composable utilizing Material 3 `AlertDialog`.
- Maps `OutputFolderValidation` results to matching string resources.

### 2. String Resources in `strings.xml`
- Add validation message strings for `NotReachable`, `PermissionRevoked`, `NotWritable`, and `InsufficientSpace`.

### 3. ViewModel Wiring
- Inject `OutputFolderValidator` in:
  - `SettingsViewModel`
  - `SplitConfigViewModel`
  - `MergeConfigViewModel`
- Add `validationResult: StateFlow<OutputFolderValidation?>` state flow.
- Add `onPickFolderAgain()` and `dismissValidation()` triggers.

### 4. Screen Wiring
- Collect `validationResult` flow.
- Display `FolderValidationDialog` when validation result is non-null and not `Ok`.
- Launch tree folder picker upon "Pick again" button action.

### 5. Unit Tests
- `SettingsViewModelTest`: mock validator, assert proper state emissions.
- `SplitConfigViewModelTest`: mock validator, assert proper state emissions.
- `MergeConfigViewModelTest`: mock validator, assert proper state emissions.
